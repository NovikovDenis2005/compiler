# MiniJS Compiler

Учебный компилятор подмножества JavaScript (с уклоном в Pascal-подобный синтаксис
для арифметики: `div`, `mod`). Реализация — на чистом Java 21, без ANTLR и
других генераторов.

## Что внутри

Полный конвейер:

```
исходник.js
   │
   ▼
[1] Лексер           → токены
[2] Парсер           → AST (records)
[3] Семантика        → таблица символов, типы, диагностика
[4] Оптимизации AST  → свёрнутое и упрощённое дерево + статистика
[5] Интерпретатор    → выполнение AST
```

## Запуск

Сборка:

```sh
mvn -q -DskipTests package
```

Запуск интерактивный (предложит выбрать пример из `examples/`):

```sh
java -jar target/miniJS-1.0-SNAPSHOT.jar
```

Запуск с конкретным файлом:

```sh
java -jar target/miniJS-1.0-SNAPSHOT.jar examples/06_constant_folding.js
```

## Архитектура

Каждый этап — отдельный пакет с одной точкой входа:

| Этап | Пакет        | Класс                       |
|------|--------------|-----------------------------|
| 1    | `lexer`      | `Lexer`                     |
| 2    | `parser`     | `Parser`                    |
| 3    | `semantic`   | `SemanticAnalyzer`          |
| 4    | `transform`  | `AstTransformer`            |
| 5    | `interpreter`| `Interpreter`               |

AST — `sealed interface ASTNode` с `record`-вариантами (`ast/ASTNode.java`).
Диагностика лексера, парсера и семантики идёт через `diagnostic/Diagnostic.java`
с подсветкой места ошибки в исходнике.

## Подмножество JavaScript

| Группа                 | Что поддерживается                                              |
|------------------------|-----------------------------------------------------------------|
| Типы данных            | number, string, boolean, null, undefined, массивы, объекты      |
| Объявления             | `let`, `function`                                               |
| Арифметика             | `+ - * / div mod`                                               |
| Сравнения              | `== != < > <= >=`                                               |
| Логика                 | `&& || !`                                                       |
| Управление потоком     | `if/else`, `while`, `for`, `break`, `continue`, `return`        |
| Ввод-вывод             | `console.log(...)`                                              |
| Доступ к данным        | `obj.prop`, `arr[i]`                                            |
| Строковая конкатенация | `+` с JS-семантикой (строка с числом и т.д.)                    |

Намеренно НЕ поддерживается: классы, замыкания над `this`, `var`, `const`, switch,
try/catch, шаблонные строки, генераторы, async/await — всё, что выходит за рамки
учебной задачи.

## Оптимизации (3-я аттестация)

Все оптимизации работают на AST (один проход сверху вниз), результаты учитываются
в `OptimizationStats` и печатаются в конце этапа 4.

### Свёртка константных выражений

```js
let a = 2 + 3 * 4;   //  -> 14
let g = 5 < 10;      //  -> true
```

### Constant propagation через let

Известное литеральное значение переменной подставляется на её место в выражениях,
если переменная больше нигде не переопределяется.

```js
let x = 10;
let z = x + 5;       //  -> 15  (x подставлен, потом свёрнут)
```

### Алгебраические тождества

Применяются только когда «не-литеральная» часть синтаксически выводимо числовая,
иначе JS-семантика `+` (строка с числом — это конкатенация) ломала бы поведение.

```js
(a * b) + 0    ->  (a * b)
1 * (a * b)    ->  (a * b)
0 * (a * b)    ->  0
(a < b) && false -> false
!!(a < b)        -> (a < b)
```

### Удаление мёртвых веток

```js
if (true)  { A } else { B }   -> A
while (false) { ... }          -> вычёркивается
```

### Недостижимый код после return/break/continue

```js
function f(n) {
    return n;
    console.log("сюда не дойдём");   // выбрасывается
}
```

### Удаление неиспользуемых переменных

Объявление `let x = ...` удаляется, если `x` нигде не читается, не пишется
и инициализатор без побочных эффектов (вызов функции считается побочным эффектом —
такое не удаляем).

## Ограничения

- Целевого кода нет — выполняется AST-интерпретатором.
- Алгебраические правила требуют статически выводимого числового или булева
  операнда: `(a*b) + 0` упрощается, а `x + 0` для `x: Identifier` — нет
  (иначе сломается JS-конкатенация `"abc" + 0`).
- Шейдинг имён в анализе неиспользуемых переменных не учитывается.
